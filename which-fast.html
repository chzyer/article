哪个快?

February 28, 2013

在处理一个问题, 用一个 updated 变量来表示是否进入了 for 循环

updated := false
for i:=0; i<1000000; i++ {
    updated = true
}

然后就会有一个问题, 这样每次都赋值一次会不会太浪费资源, 于是又有了下面的写法:

updated := false
for i:=0; i<1000000; i++ {
    if ! updated {
        updated = true
    }
}

然后是哪种做法快了呢? 无非是在比 if 的执行速度快还是赋值 true 比较快, 很简单, 计时

t := time.Now()
a := false
for i:=0; i<1000000; i++ {
    if ! a {
        a = true
    }
}
_ = a // 为了让go编译通过
println(time.Now().UnixNano() - t.UnixNano())

得到用时是 864000

t := time.Now()
a := false
for i:=0; i<1000000; i++ {
    a = true
}
_ = a
println(time.Now().UnixNano() - t.UnixNano())

用时是446000
